# 产品需求文档（PRD）V2.0：盘串 · 每日心珠练习

**版本**：V2.0 | **状态**：修订稿（基于V1.0讨论重构） | **目标市场**：美国 iOS 应用商店

> 本版本相较V1.0的核心调整：**去除应用内一切社交/社群结构**，社交边界严格收窄为"生成分享图片，由用户通过自己的社交账号对外分享"；**术语去宗教化**，用美式心灵成长/正念文化中已有的世俗类比替代直译的佛教术语；产品闭环从"参-录-缘"三阶收窄为"练-录"两阶核心 + "传"这一单向对外增长机制；技术栈与功能范围相应简化，以更贴合"轻量级"定位与App Store审核现实。

## 一、产品概述

### 1.1 产品名称
**盘串**（英文名：*Beads*）

命名逻辑不变：呼应"盘手串"这一兼具仪式感与日常陪伴感的世俗行为，强调"日日摩挲、渐渐温润"的个人心境养成，而非宗教修行。

### 1.2 产品定位
> **"一个从东方哲学与现代正念心理学中提炼智慧、以个人化每日练习为核心的轻量级心灵成长工具。"**

它不依赖群体归属感来驱动留存和传播——用户的练习和记录完全私有，唯一"走出App"的内容是用户自己主动生成、自己决定分享去向的一张图片。App不构建任何"被围观、被回应"的结构。

### 1.3 核心理念（产品价值观）
1. **日拱一卒**：每日一练、一录，微行动带来内心渐变
2. **反焦虑**：不设排行榜、不比拼、不引入任何群体可见的比较场景
3. **完全私有**：练习记录和心得默认且永久仅自己可见，没有"公开"选项
4. **无信仰门槛**：智慧内容源自东方哲学与现代心理学，术语世俗化，不预设任何宗教身份认同
5. **分享是权利，不是功能**：应用不主动构建社交场域，只在用户主动选择时，提供一张可以带去任何平台的漂亮图片

### 1.4 一句话打动用户
> "每天3分钟，盘一串属于你自己的内心念珠。"

## 二、目标用户画像

### 2.1 核心用户群（T1）

| 维度 | 描述 |
| :--- | :--- |
| **人口特征** | 25～45岁，男女比例约4:6，居住在城市（纽约、洛杉矶、旧金山等），本科以上学历 |
| **收入水平** | 年收入5万～12万美元 |
| **兴趣标签** | 正念冥想、瑜伽、自我成长、文学诗歌、极简主义、东方哲学 |
| **已有行为** | 使用过Calm/Headspace/Insight Timer/Day One，对社交媒体本身有一定疲惫感 |
| **核心诉求** | 想要一种**更轻盈、更私密、有文化厚度**的精神修养方式，不希望被拉入新的"圈子"或"社群" |

**关键洞察**：这批用户下载正念类App的动机之一恰恰是"想从feed里逃出来"。任何应用内可见的群体结构（哪怕匿名、哪怕温和）都可能与他们的核心诉求相冲突。这是V2.0砍掉共修圈类功能的用户心理依据，不只是审核层面的考量。

### 2.2 外围用户群（T2）
- 对东方哲学好奇的泛文化人群
- 正在经历生活/工作压力，寻求私密出口的普通白领
- 已有佛教信仰基础，但不愿使用严肃宗教App或宗教社群App的人群

### 2.3 用户旅程示例

> - 21:00 💡 睡前看了眼本周的心情小结，没有任何"别人怎么样了"的信息需要处理

## 三、核心功能需求（两阶核心 + 一项对外增长机制）

### 第一阶：练——每日心珠练习
*（原"参"，术语调整为更贴近西方正念文化的"practice"概念）*

| 功能模块 | 详细需求 | 优先级 |
| :--- | :--- | :--- |
| **每日心语推送** | 每日早上8:00（用户可自定义）推送一条"东方哲学转译金句 + 今日练习"，内容约40～80字，英文原创措辞（非机器翻译） | P0 |
| **一键"完成"交互** | 用户阅读后点击"完成"（英文：Practice / Done），记录当日练习；仅需一键，门槛极低 | P0 |
| **主屏/锁屏Widget** | 今日心语直接展示在主屏或锁屏小组件，交互式Widget支持不打开App直接点击"完成"（iOS 17+ App Intents） | P0 |
| **练习连续记录** | 连续练习天数可视化为"心珠"串——每7天成珠一颗，颜色随连续天数渐变 | P0 |
| **错过补练机制** | 允许补签最近3天（每日限补1次），维持连续记录；补签需完成简短反思练习 | P1 |
| **历史练习回顾** | 日历视图查看所有练习记录 | P1 |
| **Siri Shortcuts / App Intents** | "嘿Siri，我完成今日练习了"，语音一键打卡 | P1 |
| **Apple Watch端** | 简易complication显示今日心语摘要 + "完成"按钮 | P2 |

**内容库规格**：
- 上线时预制365条"心语+练习"（至少1年量）
- 内容来源策略：禅宗公案、老庄、慧能、铃木俊隆等经现代语言转译；现代正念作者（需确认版权）；不直接引用宗教经文原文
- **不出现的词汇**：业力、轮回、供养、功德、开悟、修行、法门等具体宗教术语；改用practice、awareness、stillness、intention等世俗心理学词汇
- 内容须由英语母语撰稿人/文化顾问审阅，避免直译腔和潜在的文化挪用争议

### 第二阶：录——每日心迹留存
*（保持V1.0设计，完全私有，无可见性设置）*

| 功能模块 | 详细需求 | 优先级 |
| :--- | :--- | :--- |
| **心得记录** | 60字以内短语记录，保持轻量 | P0 |
| **心情标记** | 😌平静 / 😤烦躁 / 🙏感恩 / 😢低落 / 🔥焦虑 / 🌱成长，选1～2个 | P0 |
| **心迹集归档** | 按月归档，自动生成情绪关键词云封面 | P1 |
| **导出为小书** | 任意月份心迹集生成图文卡片序列，保存为图片/PDF | P2 |
| **记录自由** | 可只标心情不写文字，或跳过记录，不强制 | P0 |
| **无社交属性** | 所有记录默认且永久仅自己可见，**不提供"公开"选项**，无需可见性权限设置 | P0 |

### 第三阶：传——单向对外分享（原"缘"，不构成产品内社交结构）

> **设计原则**：应用内不存在任何"其他用户可见"的内容或结构。用户能做的唯一"向外"动作，是生成一张图片，然后自己决定要不要分享、分享去哪。

| 功能模块 | 详细需求 | 优先级 |
| :--- | :--- | :--- |
| **心珠卡片（原"智慧涟漪"）** | 一键将任意心得/心语生成精美分享卡片；5～8种视觉模板（水墨风/极简白/渐变夕阳等）；卡片角落带低调水印"Beads App"及下载二维码/链接；分享至iOS系统分享面板（Instagram/Twitter/TikTok等） | P0 |
| **善意树（原"善缘树"，个人化邀请统计）** | 用户拥有一棵私有的"善意树"，仅自己可见；通过分享链接邀请朋友，朋友完成7天练习后树上开一朵花；**受邀者列表仅自己可见，不构成任何用户间可见的社群**，纯粹是个人成就可视化 | P1 |
| **邀请深链接** | 分享链接通过Universal Links跳转App Store对应页面；可选：制作App Clip供受邀者免安装体验3秒版"今日心语+完成"，提升邀请转化率 | P2 |

**明确不做的功能**（相较V1.0的关键取舍）：
- ❌ 共修圈 / 圈子加入机制 / 圈内信息流
- ❌ 匿名心得广场 / 任何"看到他人内容"的场景
- ❌ 随喜点灯 / 任何用户对用户的互动
- ❌ 心得的"公开/仅圈内/仅自己"三级可见性设置
- ❌ 评论区、私信、点赞

### 跨阶通用功能

| 功能模块 | 详细需求 | 优先级 |
| :--- | :--- | :--- |
| **用户系统** | **Sign in with Apple（必须，作为与其他登录方式对等或更优先的选项，符合Apple Guideline 4.8）** + 邮箱 + Guest模式（本地存储，登录后同步） | P0 |
| **数据同步** | 采用 **CloudKit 私有数据库**（每用户数据仅归属自己，不涉及跨用户共享数据库层），跨设备同步iPhone/iPad/Watch数据一致 | P0 |
| **推送通知** | 可自定义推送时间（默认8:00）；"今日心语"推送；"连续练习提醒"（20:00温和提醒）；**推送内容仅涉及用户自己的数据，不推送"他人动态"** | P0 |
| **隐私与数据** | 一键导出所有个人数据（GDPR/CCPA合规）；随时删除账户及所有数据；因无跨用户数据暴露，隐私合规范围显著收窄 | P0 |

## 四、非功能性需求

| 需求项 | 要求 |
| :--- | :--- |
| **平台** | iOS 16.0+，**MVP阶段仅支持iPhone**（不勾选iPad支持，避免因未做专门适配被审核拒绝），iPad适配放入V2.0规划 |
| **包体积** | 目标 < 50MB |
| **加载速度** | 首页加载 < 1.5秒 |
| **离线可用** | 已推送的"今日心语"和已记录的心得可离线查看 |
| **国际化** | 英文为主（原创英文文案，非机器翻译），内置多语言支持（未来拓展） |
| **无障碍** | 支持VoiceOver，字体大小跟随系统 |
| **技术栈建议** | SwiftUI（UI层）+ **CloudKit（数据与同步，私有数据库即可满足全部需求，不需要Firebase/Supabase等第三方跨用户BaaS）**+ WidgetKit + App Intents + StoreKit 2 |

**技术栈简化说明**：由于V2.0不存在任何跨用户可见数据（无圈子、无信息流、无点灯计数），后端复杂度大幅降低。CloudKit私有数据库天然贴合"每用户数据仅归属自己"的场景，免费额度充足，且作为Apple原生方案，隐私合规主体更单一、审核沟通成本更低。仅当未来确有Android版本规划时，才需要引入跨平台BaaS。

## 五、Apple原生轻量能力专项（新增章节）

这类"每日打卡+心语记录"产品，最适合靠iOS系统级能力做留存，且开发成本远低于自建社交功能：

| 能力 | 应用场景 | 建议阶段 |
| :--- | :--- | :--- |
| **主屏Widget** | 今日心语 + 心珠进度展示 | V1.0 |
| **交互式锁屏/主屏Widget** | 免开App直接点击"完成"（App Intents，iOS 17+） | V1.0～V1.1 |
| **Siri Shortcuts / App Intents** | 语音一键打卡，契合"极低门槛"理念 | V1.1 |
| **Apple Watch App** | 简易complication显示心语摘要+完成按钮 | V1.1～V1.2 |
| **App Clip** | 受邀者免安装体验，提升邀请转化率 | V1.2（增长阶段） |
| **StandBy / 灵动岛** | 待机模式下展示今日心语（可选） | V2.0探索 |

这些功能对D1/D7留存和邀请转化的提升，预计优于任何应用内社交机制，且不产生UGC审核负担。

## 六、运营与内容需求

### 6.1 内容生产体系
- 内容团队（初期创始人兼任）：1～2人负责每日心语选编与练习设计，**须配备英语母语审校/文化顾问**，避免翻译腔和文化敏感度问题
- 内容来源：70%经典转译（禅宗公案/道家经典/铃木俊隆等）+ 20%现代正念改编（确认版权）+ 10%后期用户投稿精选（不涉及站内可见，仅作为官方内容库的补充素材来源，需人工审核后收录，不构成用户对用户的展示）
- 内容审核标准：不涉及宗教传教；不涉政治；不医疗宣称；不含"业力/功德/开悟"等具体教义词汇

### 6.2 种子用户获取（MVP阶段）
- Product Hunt首发
- r/Meditation、r/Buddhism、r/Mindfulness、r/Zen等Subreddit（遵守社区规则，注意措辞去宗教化以降低争议风险）
- "7天放下焦虑"免费邮件课程引流
- 联系正念/禅修类KOL，提供终身免费码换取评测

### 6.3 增长引擎
- **心珠卡片外部分享**：唯一的病毒传播路径，需确保卡片设计足够精美、水印/二维码足够低调但有效（此路径承担了原本"共修圈"部分的传播职能，需要重点打磨）
- **善意树邀请**：无物质奖励，纯粹的个人成就感驱动
- **节日营销**：心理健康月（5月）、新年季（1月）、感恩节等

### 6.4 用户留存策略
- 第3天：推送"你已经练习3天了，第一颗心珠正在成形"
- 第7天：推送"恭喜！你的第一颗心珠已圆满"+ 生成"首周心迹卡"
- 第30天：推送"你已经练习一个月了"+ 月度报告
- 流失召回：连续3天未练习，推送"心珠还在等你"+ 一条精选心语

## 九、风险与规避

| 风险类型 | 具体描述 | 应对策略 |
| :--- | :--- | :--- |
| **内容风险** | 术语翻译可能被视为文化挪用；措辞不当可能引发"东方主义猎奇化"争议 | 所有术语世俗化改造（练习/心珠/善意树，不用mala/karma/merit直译）；英语母语文化顾问审阅全部文案 |
| **审核风险** | App Store对宗教类应用及UGC应用审核较严 | 分类选择"Lifestyle"；**因彻底移除应用内社交结构，UGC相关审核风险已从根源消除**；截图/描述不出现宗教符号 |
| **登录合规风险** | 提供第三方登录但未提供Sign in with Apple会被拒审 | Sign in with Apple设为P0，与其他登录方式对等展示 |
| **订阅合规风险** | 促销价/试用条款展示不清晰会被拒审 | 严格遵循StoreKit价格阶梯展示要求，配备恢复购买按钮 |
| **竞争风险** | 美区正念类App竞争激烈 | 差异化定位——"完全私有的心灵练习工具"，明确区隔于依赖社群/排行榜的竞品 |
| **留存风险** | 失去社交回访钩子后留存承压 | 依靠心珠可视化、Widget/Watch原生体验、内容质量本身驱动留存 |
| **合规风险** | CCPA/GDPR数据隐私合规 | CloudKit私有数据库天然缩小合规范围；提供数据导出和删除功能 |

## 十、成功指标（KPI）

| 指标 | 定义 | MVP目标 | V1.0目标 |
| :--- | :--- | :--- | :--- |
